day-boundary

Boundary Windows For Real Operational Days

Assign timestamps to operational windows when midnight breaks your logic. Use day-boundary when reporting days, service windows, shift cycles, or cutoffs do not align with the calendar day.

Examples Tour Usage Guide API Guide

Golden Path

How Most Applications Use It

  1. Define a boundary strategy.
  2. Resolve the surrounding [start, end) window for an exact timestamp.
  3. Use window.start and window.end for querying, grouping, and reporting.

If you only start with one function, start with getWindowForInstant(...).

Critical Cases

What This Repo Covers Explicitly

  • Non-midnight and overnight operational days
  • Spring-forward gaps and skipped local times
  • Fall-back ambiguity and repeated local timestamps
  • Elapsed versus wall-clock duration drift
  • Cross-zone business-date divergence near the International Date Line

Quick Start

import { Temporal } from '@js-temporal/polyfill';
import { FixedTimeBoundaryStrategy, getWindowForInstant } from 'day-boundary';

const strategy = new FixedTimeBoundaryStrategy({
  timeZone: 'Europe/London',
  boundaryTime: '09:00',
});

const window = getWindowForInstant(Temporal.Now.instant(), strategy);

console.log(window.start.toString());
console.log(window.end.toString());

This package is ESM-only and the typed API is Temporal-only.
